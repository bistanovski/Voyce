Detailed Description

Broadcast element is built as signal dispatcher, which are used between different elements and objects in Voyce application. Each item/object instance can "subscribe" to the signals provided in Broadcast, and on new emit, they can handle the processing part in their part of code blocks. This is good initial approach for centralized registrar of events for simple applications.

Signal Documentation

addToPlaylist(var = episodeData)

Emitted when episodeData object need to be added in the current playlist


clearAndSaveSettings()

Emitted when settings need to be updated


clearCache()

Emitted when network cache clearing is needed


clearPodcasts()

Emitted when new podcast data has to be cleared


clearTags()

Emitted when podcast tags stored data has to be cleared

Note: tags, refer on the same thing as Category in which podcasts are grouped


downloadEpisode(var = episodeData)

Emitted when selected episodeData object is scheduled for downloading


downloadFinished(var = episodeData)

Emitted when episodeData object has finished downloading


fetchPodcastTags(var = numOfTags)

Emitted when new podcast tags need to be fetched, with numOfTags total tags


fetchPodcastsForTag(var = tag, var = numOfPodcasts)

Emitted when new search for podcasts based on tag and total numOfPodcasts results


playEpisode(var = episodeData)

Emitted when selected episodeData object is scheduled for playing


removeFromDownloads(var = episodeIndex, var = localUrl)

Emitted when episode with episodeIndex located on localUrl need to be deleted


removeFromPlaylist(var = episodeIndex)

Emitted when episode with episodeIndex need to be removed from the current playlist
